# Proposal: Unify Daily Usage Display

## Summary
将 tenant-select-card 中的「今日费用」和「今日用量」合并为单一标签「今日消耗」，同时展示费用（金额）和用量（tokens）两种单位，避免用户误解为两个独立概念。

## Motivation
当前设计将同一数据（今日消耗）拆分为两行展示：
- 「今日费用」显示金额（如 $1.23）
- 「今日用量」显示 token 数（如 1,234 tokens）

这种分离可能让用户误以为是两个不同的指标，实际上它们是同一消耗数据的不同计量单位。统一展示可以：
1. 减少认知负担，明确表达「一个概念，两种单位」
2. 节省垂直空间，使卡片更紧凑
3. 提供更直观的数据对照

## Proposed Design

### 视觉方案
将两行合并为一行，使用单一标签 + 双值展示：

```
今日消耗    $1.23 / 1,234 tokens
```

或者使用分隔符样式：
```
今日消耗    $1.23 · 1,234 tokens
```

### 数据缺失处理
- 两者都无数据：显示 `–`
- 仅有费用：显示 `$1.23`
- 仅有 tokens：显示 `1,234 tokens`
- 两者都有：显示 `$1.23 / 1,234 tokens`

## Scope
- 仅影响 `components/biz/tenant-select-card/index.tsx`
- 不涉及数据层或 API 变更
- 纯 UI 展示层调整

## Risks & Mitigations
| Risk | Mitigation |
|------|------------|
| 单行过长导致换行 | 使用紧凑格式，必要时截断 token 数 |
| 用户习惯变化 | 变更较小，标签语义更清晰 |

## Design Decisions
- 分隔符：使用 `/`
- 大数值格式：千分位分割（如 1,234,567 tokens），不使用缩写
