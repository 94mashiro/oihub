# Delta: popup-navigation

## MODIFIED Requirements

### Requirement: Settings Page Layout
The settings page SHALL display a header with title and back button, and its content sections that are presented as cards SHALL use design tokens consistent with main screen cards (rounded corners, compact padding, border/shadow, and hover/interactive states where applicable).

#### Scenario: Settings page card consistency
- **WHEN** user navigates to the settings page
- **THEN** any card-like sections (e.g., per-tenant alert configuration items) visually match the main screen card style
